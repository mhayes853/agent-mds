# AGENT MDS

Contains some common base `AGENTS.md` files that I tend to use. They can be modified on a per-project basis accordingly.

In particular, I try to focus on dispelling anti-patterns at a fine-grained level rather than telling the agent what the project architecture is, and how to work within it. Generally, it's quite easier to steer an agent at prompt time through an architecture, rather than try to steer it to not mess up the small things.

The conventions are mostly my personal preferences, I don't expect you to like them.
